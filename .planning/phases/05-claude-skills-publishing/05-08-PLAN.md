**Step 1: Read 05-VERIFICATION.md and extract all failures.**

Parse the Failure Summary section. Group by severity:

- Critical failures: MUST fix. Block all skills.
- Major failures: MUST fix. Block affected skill.
- Minor failures: SHOULD fix. No block but fix before publish.

**Step 2: For each failure, read the corresponding failure report** from the results file. Each failure report has L1/L2/L3 fix proposals. Apply fixes using this priority:

- Prefer L1 (patch) if the fix is sufficient and low-risk
- Use L2 (structural) if L1 would be fragile or if multiple related failures share a root cause in shared/ files
- Use L3 (rethink) only if L1/L2 are insufficient

**Step 3: Apply fixes to the actual skill/shared files.**

For each fix:

- Read the file being modified (already in read_first)
- Make the specific edit described in the failure report
- If fixing a shared/ file, check impact radius — the failure report lists which other skills are affected
- Do NOT change things that are passing — targeted surgery only

Common fix patterns expected:

- Missing instruction in SKILL.md (e.g., no explicit handling for malformed YAML) -> add instruction paragraph
- Voice violation (em-dash, prohibited phrase) -> search and replace
- Missing CTA or incorrect CTA text -> add/update CTA line
- Scoring edge case not covered -> add edge case handling instruction
- Safety protocol gap -> add MIND-SAFE instruction for specific scenario

**Step 4: Re-test all fixed issues.**

Create `.planning/phases/05-stress-testing/results/retest-results.md`. For each fixed failure:

- Re-run the exact test that failed (using same test ID and criteria from eval-cases/)
- Record PASS/FAIL with evidence
- If still FAIL, escalate to next fix level (L1 failed -> try L2)

Format:

```
### [TEST-ID] — RETEST
**Original failure:** [brief]
**Fix applied:** [what was changed, which file, which line]
**Retest result:** PASS / FAIL
**Evidence:** [line numbers or text proving fix works]
```

**Step 5: Update 05-VERIFICATION.md.**

Update the 6x5 matrix cells that changed from FAIL to PASS. Update the Failure Summary. Recalculate the verdict.

If all Critical and Major failures are fixed and retests pass, update verdict to: CONDITIONALLY READY (pending Manual Review).

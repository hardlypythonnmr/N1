**gpts/busca-adentro/instructions.md** — the GPT system prompt (per D-01, D-02, D-03):
Target: ~1,800 tokens / ≤1,400 words. Strict. Everything that does not fit goes to knowledge files.
Content structure (in this order):

1. Identity block (3 lines): "Soy N1 Busca Adentro, una herramienta de autoconocimiento profesional para LATAM. Uso investigación validada. Hablo en español colombiano con voseo (vos tenés, contame, hablás)."
2. Safety disclaimer — verbatim Spanish version from shared/safety-protocol.md Section 1. Do not paraphrase. (per D-07, D-02 priority: Safety first)
3. MIND-SAFE check instruction (2-3 lines): check every user message for Tier 1/Tier 2/Tier 3 signals; Tier 1 = stop and use crisis response; reference methodology.md for tier definitions.
4. Profile import step: ask if user has N1 YAML profile; if yes extract completed dimensions; if no proceed to triage.
5. Triage (3 numbered questions exactly as in SKILL.md): situation, career direction, urgency. Routing table: Crisis → values quick; Exploration → all 4 quick or deep; Growth → strengths first. Save triage to YAML.
6. Mode selection: quick (5-10 min/dimension) or deep (30-45 min/dimension); default quick; crisis always quick.
7. Dimension flow — 4 numbered sections, each with: opener quote, quick mode (5 questions listed by number), soft gate to next dimension, YAML output block. Keep each dimension to 5-7 lines. Full exercises and frameworks are in exercises.md and methodology.md.
8. Full profile YAML output block (after all dimensions): version: "1.0", triage, values, strengths, identity, purpose structure — identical to Profile Interchange Format v1.
9. Voice rules (4 lines): voseo, no "networking"/"marca personal"/"agregar valor", cite evidence, no motivational fluff.
10. Cross-GPT CTA (2 lines per D-08): "¿Querés aplicar esto? Probá N1 Construye Afuera GPT. ¿Querés primero un snapshot rápido? Probá N1 Compass GPT."

Compression strategy (per D-01):

- KEEP in instructions: flow steps, triage questions, YAML output blocks, safety disclaimer, voice rules
- MOVE to knowledge files: framework theory (Schwartz, Bandura, Ibarra, Frankl), Deep Mode step-by-step, exercise lists, citation bank, full YAML schema

- Purpose: Frankl 3 meaning sources; MLQ quadrants (presence vs search); LATAM calibration (high MLQ-Search is normative)
  Include LATAM calibration note at top: "Control subscale runs lower in collectivist cultures — scores of 2-3 are normative in Colombia, not a deficit."

Content structure (in this order):

3. MIND-SAFE check (3 lines): check every user message; Tier 1 = stop + crisis response; Tier 2 = warm referral then continue; job search context = Tier 2 possible (estrés financiero, duelo por despido).
4. Profile check: "¿Tenés un perfil N1? Podés pegarlo para agregar análisis de alineación de valores. Si no, igual analizo la oferta — funciona igual."
5. Input request: "Pegá el texto completo de la oferta, o describímela: empresa (o si es confidencial), rol, salario si aparece, fecha de publicación, cualquier detalle relevante." Add LATAM note: "En LATAM es común que no incluyan salario — ya está considerado en el scoring."

   | Categoría | Max pts | Criterio |
   Include all 6 rows: Antigüedad (20), Transparencia salarial (20), Especificidad del rol (15), Presencia de la empresa (15), Patrones de lenguaje (15), Señales de reposteo (15).
   Plus adjustment factors: requisitos irreales (-5), urgencia vacía (mention), solo email (mention).
7. Score tiers — 4 tiers verbatim from SKILL.md: 80-100, 60-79, 40-59, 0-39 with verdicts.
8. Output format — exact format from SKILL.md: "Ghost Score: [X]/100 — [veredicto]" + signal table with ✓/⚠/✗.
9. Values alignment section (only if profile provided): read values.core_values and values.bulls_eye.work; estimate alignment 1-10; if bulls_eye.work < 5, name the misalignment explicitly.
10. Shareable output: offer LinkedIn summary; CRITICAL rule verbatim: "NUNCA incluir el texto de la oferta pegado por el usuario en el contenido compartible."
11. CTAs (per D-08): after any session → suggest N1 Compass for career readiness check; reference ghost-check web tool for visual scorecard.
12. Reflexion checklist (inline, silent): 5-item checklist from SKILL.md, executed internally before each response.

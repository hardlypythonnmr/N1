Create web/assessment-engine.js — the shared JS utility module loaded by every tool via &lt;script src="../assessment-engine.js"&gt; (or &lt;script src="assessment-engine.js"&gt; from landing). Pure vanilla JS, no dependencies, no ES modules (tools use script tag imports).

Expose everything on a global `N1` object: `window.N1 = {}`.

Functions to implement:

**1. N1.createAssessment(config)**
Accepts config: { containerId, questions: [{id, text, type: 'likert5'|'likert7'|'choice', options?}], onComplete: fn }
Renders one-question-per-screen flow inside the container element.
State: { current: 0, answers: {} }
Each screen: progress bar (current/total %), question number ("Pregunta 3 de 12"), question text, answer inputs, back/next buttons.
Keyboard nav (per D-06): Enter or ArrowRight advances, ArrowLeft goes back.
Back button (per D-07): visible from question 2 onward. Preserves prior answers when navigating back.
Auto-advance on Likert selection: when user clicks/selects a Likert option, auto-advances after 400ms delay (Typeform pattern).
On last question completion: calls onComplete(answers).

**2. N1.loadProfile()**
Reads localStorage key 'n1-profile'. Returns parsed JSON object or null if missing/invalid.
Handles JSON.parse errors silently (returns null on failure).

**3. N1.saveToProfile(section, data)**
Reads existing profile (or creates fresh one from template structure).
Deep-merges data into profile[section].
Sets profile.updated to new Date().toISOString().
If profile.created is null, sets it.
Writes back to localStorage under 'n1-profile'.
Returns the updated profile object.

**4. N1.generateYAML(profile)**
Converts profile JSON to YAML string manually (no library — pure string construction).
Must include version: "1.0" header comment block.
Format: proper YAML indentation, null for null values, arrays with - prefix.
Returns YAML string.

**5. N1.exportYAML(section, data)**
Calls saveToProfile, then generateYAML on full profile.
Creates a Blob, triggers download as 'n1-profile.yaml'.
Also copies YAML to clipboard via navigator.clipboard.writeText (with fallback for older browsers).

**6. N1.generateScorecard(config)**
config: { title, scores: [{label, value, max}], interpretation, chartType: 'bar'|'radar'|'quadrant' }
Creates offscreen Canvas element at 1200x628px (per D-09).
Draws: N1 orange header band (top 80px), title in Space Mono white, scores visualization, interpretation text (DM Sans), footer "N1 — el sistema eres tu" in Space Mono gray.
Returns data URL (PNG).

**7. N1.downloadScorecard(dataUrl, filename)**
Creates link element, sets href to dataUrl, triggers download.

**8. N1.copyLinkedInText(text)**
Copies text to clipboard. Returns Promise.
Use navigator.clipboard.writeText with try/catch fallback using execCommand.

**9. N1.getSafetyDisclaimer()**
Returns the verbatim Spanish disclaimer HTML string (from safety-protocol.md).
Used by tools to inject disclaimer into their DOM.

**10. N1.initLocalStorageSafe()**
Wraps localStorage access in try/catch. Returns boolean: true if available, false if not.
Call on page load. If false, show banner: "Tu navegador no permite guardar datos localmente. Los resultados se mostrarán pero no se guardarán."

Error handling (per coding-style.md): never silently swallow errors. Log to console with context. Show user-visible fallback only for actionable errors.

Immutability (per coding-style.md): all state operations return new objects, never mutate existing ones.
